40 ### <a name="introduction">Introduction</a>
41 `fuzzylite` is a free and open-source fuzzy logic control library programmed in C++ for multiple platforms (Windows, Linux, Mac, iOS). Its goal is to allow you to easily create fuzzy logic controllers in a few steps utilizing object-oriented programming without requiring any third-party libraries.
42

58 ### <a name="features">Features</a>
59 
60 **Controllers** *Types* (5) Mamdani, Takagi-Sugeno, Larsen, Tsukamoto, Inverse Tsukamoto
61 
62 **Linguistic terms** *Basic* (4) Triangle, Trapezoid, Rectangle, Discrete. *Extended* (9) Bell, Cosine, Gaussian, Gaussian Product, Pi-shape, Sigmoid Difference, Sigmoid Product, Spike. *Edges* (4) Concave, Ramp, Sigmoid, S-shape, Z-shape. *Functions* (3) Constant, Linear, Function
63 
64 **Conjunction and Activation** *T-Norm* (7) minimum, algebraic product, bounded difference, drastic product, einstein product, hamacher product, nilpotent minimum.
65 
66 **Disjunction and Accumulation** *S-Norm* (8) maximum, algebraic sum, bounded sum, normalized sum, drastic sum, einstein sum, hamacher sum, nilpotent maximum.
67 
68 **Defuzzifiers** *Integral* (5) centroid, bisector, smallest of maximum, largest of maximum, mean of maximum, *Weighted* (2) weighted average, weighted sum.
69 
70 **Hedges** *Types* (6) any, not, extremely, seldom, somewhat, very.
71 
72 **Import** *Types* (3) FuzzyLite Language `fll`, Fuzzy Inference System `fis`, Fuzzy Control Language `fcl`.
73 
74 **Export** *Types* (6) `C++`, `Java`, FuzzyLite Language `fll`, FuzzyLite Dataset `fld`, Fuzzy Inference System `fis`, Fuzzy Control Language `fcl`.
75 
76 **Examples** (30+) of Mamdani, Takagi-Sugeno and Tsukamoto controllers from `fuzzylite`, Octave and Matlab, each included in the following formats: `C++`, `Java`, `fll`, `fld`, `fis`, and `fcl`.

80 ### <a name="example">Example</a>
81 
82 ```cpp
83 #include "fl/Headers.h"
84 
85 int main(int argc, char* argv[]){
86  using namespace fl;
87  Engine* engine = new Engine("simple-dimmer");
88 
89  InputVariable* ambient = new InputVariable;
90  ambient->setName("Ambient");
91  ambient->setRange(0.000, 1.000);
92  ambient->addTerm(new Triangle("DARK", 0.000, 0.500));
93  ambient->addTerm(new Triangle("MEDIUM", 0.250, 0.750));
94  ambient->addTerm(new Triangle("BRIGHT", 0.500, 1.000));
95  engine->addInputVariable(ambient);
96 
97  OutputVariable* power = new OutputVariable;
98  power->setName("Power");
99  power->setRange(0.000, 2.000);
100  power->setDefaultValue(fl::nan);
101  power->addTerm(new Triangle("LOW", 0.000, 1.000));
102  power->addTerm(new Triangle("MEDIUM", 0.500, 1.500));
103  power->addTerm(new Triangle("HIGH", 1.000, 2.000));
104  engine->addOutputVariable(power);
105 
106  RuleBlock* ruleblock = new RuleBlock;
107  ruleblock->addRule(Rule::parse("if Ambient is DARK then Power is HIGH", engine));
108  ruleblock->addRule(Rule::parse("if Ambient is MEDIUM then Power is MEDIUM", engine));
109  ruleblock->addRule(Rule::parse("if Ambient is BRIGHT then Power is LOW", engine));
110  engine->addRuleBlock(ruleblock);
111 
112  engine->configure("", "", "Minimum", "Maximum", "Centroid");
113 
114  std::string status;
115  if (not engine->isReady(&status))
116  throw Exception("Engine not ready. "
117  "The following errors were encountered:\n" + status, FL_AT);
118 
119  for (int i = 0; i < 50; ++i){
120  scalar light = ambient->getMinimum() + i * (ambient->range() / 50);
121  ambient->setInputValue(light);
122  engine->process();
123  FL_LOG("Ambient.input = " << Op::str(light) << " -> " <<
124  "Power.output = " << Op::str(power->getOutputValue()));
125  }
126 }
127 ```

131 ### <a name="building">Building from Source</a>
132 Building from source requires you to have CMake installed.
133 
134 The files [`fuzzylite/build.bat`](/fuzzylite/build.bat) and [`fuzzylite/build.sh`](/fuzzylite/build.sh) are automatic build scripts for Windows and Unix platforms, respectively. The usage of these scripts is presented as follows.
135 
136 **Windows**
137 
138 ```bash
139 > build.bat help
140 Usage: build.bat [options]
141 where [options] can be any of the following:
142  all builds fuzzylite in debug and release mode (default)
143  debug builds fuzzylite in debug mode
144  release builds fuzzylite in release mode
145  clean erases previous builds
146  help shows this information
147 ```
148 
149 **Unix**
150 ```bash
151 $ ./build.sh help
152 Usage: [bash] ./build.sh [options]
153 where [options] can be any of the following:
154  all builds fuzzylite in debug and release mode (default)
155  debug builds fuzzylite in debug mode
156  release builds fuzzylite in release mode
157  clean erases previous builds
158  help shows this information
159 ```
160 
161 **(important)** After executing the building script, the binaries will be built and stored in the sub-folders `release/bin` and `debug/bin`.
162 
163 #### Advanced Building Options
164 For more advanced building options, please check the contents of [`fuzzylite/build.bat`](/fuzzylite/build.bat) or [`fuzzylite/build.sh`](/fuzzylite/build.sh), and the contents of [`fuzzylite/CMakeLists.txt`](/fuzzylite/CMakeLists.txt).
165 
166 The following building options are available:
167 
168 `-DFL_USE_FLOAT=ON` builds the binaries using `typedef float fl::scalar` instead of `typedef double fl::scalar` (default is OFF, i.e., double is used)
169 
170 `-DFL_BACKTRACE=OFF` disables the backtrace information in case of errors (default in Unix platforms is ON, and in Windows platforms is OFF). In Windows, the backtrace information requires the library `dbghelp`, which should be available in your system.
171 
172 `-DFL_CPP11=ON` builds `fuzzylite` utilizing `C++11` features (default is OFF, i.e., `C++98`)
173 
174 `-DCMAKE_BUILD_TYPE=[Debug|Release]` sets the mode of your build. You can only build one mode at a time with a single CMake script.

178 ### <a name="binaries">Binaries</a>
179 
180 After building from source, the following are the relevant binaries that will be created in `release` mode. In `debug` mode, binaries will append a `d` at the end of the name (e.g., `fuzzylited.dll`).
181 
182 #### Windows
183 
184 - console application: `fuzzylite.exe`
185 - shared library: `fuzzylite.dll`, `fuzzylite.lib`
186 - static library: `fuzzylite-static.lib`
187 
188 #### Linux
189 
190 - console application: `fuzzylite`
191 - shared library: `libfuzzylite.so`
192 - static library: `libfuzzylite.a`
193 
194 #### Mac
195 
196 - console application: `fuzzylite`
197 - shared library: `libfuzzylite.dylib`
198 - static library: `libfuzzylite.a`
199 
200 
201 The console application of `fuzzylite` allows you to import and export your controllers. Its usage can be obtained executing the console binary. In addition, the FuzzyLite Interactive Console is activated when exporting to `fld` without providing an output file. The interactive console allows you to evaluate any controller by manually providing the input values.
